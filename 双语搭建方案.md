# 拜茨中文站 → 中英双语全球站 搭建方案

## 核心理念

**同一个品牌（拜茨=BIOCCE），同一个域名结构，按路径区分语言。**

```
biocce.cn/zh/      → 中文（默认，国内用户）
biocce.cn/en/      → 英文（海外用户）
biocce.cn/ja/      → 日文（未来扩展）
```

## 技术架构

### 前端：Cloudflare Pages

一个CF Pages项目就够了，源码结构如下：

```
biocce-cn-site/
├── zh/                  ← 中文站（你现在做的首页+产品详情页放这里）
│   ├── index.html
│   ├── product/
│   └── ...
├── en/                  ← 英文站（复制zh的结构，内容翻译成英文）
│   ├── index.html
│   ├── product/
│   └── ...
├── shared/              ← 共享资源
│   ├── images/          ← 共用图片（Logo、产品图等）
│   ├── design-system.css
│   └── logo.png
└── _redirects           ← CF Pages路由规则文件
```

## 路线

### 第一步：中文站先上线（你现在做的阶段）
- zh/ 目录下完成所有中文页面
- 先只部署中文，跑通流程

### 第二步：加英文站
- 把 zh/ 复制成 en/
- 内容翻译成英文
- 部署时CF Pages根据URL路径 `/zh/` 或 `/en/` 自动分发

### 第三步：做语言切换
- 页面右上角加中英文切换按钮
- 用户点击自动跳转到对应语言路径

## 关键决策

| 问题 | 建议 |
|------|------|
| **SEO** | 用 `/zh/` `/en/` 路径（非子域名），Google推荐，权重集中 |
| **国内百度** | 中文内容在 `/zh/` 下，对百度友好 |
| **Google** | 英文内容在 `/en/` 下，对Google友好 |
| **备案** | .cn域名备案过的，`/zh/` 不需额外操作 |
| **部署** | 同一CF项目，git push一次更新所有语言 |

## 要不要现在就开始做？

先把zh/目录结构搭好，把你已确认的首页和产品详情模板放进去，中文站就先有了骨架，以后再逐步加英文版。
